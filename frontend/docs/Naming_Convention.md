# Naming Conventions

This guide defines nomenclature rules for HTML, CSS, JavaScript, and file assets.

## Rules
- **HTML IDs/Classes:** Always use kebab-case (e.g. `employee-table`, `.employee-table`).
- **JavaScript Variables:** Use camelCase for variables/functions (e.g., `themeManager`) and PascalCase for classes/components (e.g., `ButtonComponent`).
- **Files Naming:** Use kebab-case for CSS and JS assets (e.g., `theme-dark.css`, `app.config.js`).
- **Icons Assets:** Prefix with `ic-` (e.g. `ic-dashboard.svg`).
- **Image Assets:** Prefix with `img-` (e.g. `img-login.jpg`).
